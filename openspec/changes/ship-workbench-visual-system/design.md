## Context

整体架构明确要求 UI 达到官网级质感，但当前仓库更多是“局部优化堆叠”：不同区域使用了不同的密度、边框和强调色策略，输入区、消息区和侧栏还没有形成统一视觉语法。

## Goals / Non-Goals

**Goals:**
- 建立一套可复用的工作台视觉系统
- 让消息舞台、输入 Dock、左栏、上下文面板具备统一气质
- 将默认暗色主题和主题切换机制制度化
- 为像素审计和视觉验收提供稳定基准

**Non-Goals:**
- 不在本 change 中定义导航结构与页面层级
- 不修改 Agent / Skill / MCP 的业务契约
- 不扩展新的发布流程能力

## Decisions

1. 先定义视觉令牌，再改造组件；避免“边做页面边改 token”。
2. 主舞台遵循“阅读优先、低噪音边框、克制强调色”的原则。
3. 输入区采用单一 Dock 语义，模型、工具、发送入口纳入同一控制带。
4. 空态、加载态、失败态要使用同一视觉语法，而不是页面各自发挥。
5. Pixel 参考基线必须伴随组件体系演进持续更新。

## Risks / Trade-offs

- 视觉系统 change 很容易失控为大规模样式重写，需要坚持 token -> 组件 -> 页面 的顺序。
- 视觉统一后，原有局部定制样式可能需要删除或收敛，会引入一定回归风险。
- 若缺少稳定参考图，所谓“官网级”容易再次退化为主观判断。
