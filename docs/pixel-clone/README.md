# Pixel Clone 主计划（ChatGPT / Claude / CoWork 风格）

## 目标定义
- 目标页面：聊天页 `/`（桌面优先，含 1600x900 / 1920x1080 / 1280x800）。
- 目标标准：结构、间距、字号、对比度、交互反馈对齐“官网级聊天产品”。
- 非目标：营销官网落地页、品牌文案、图标版权素材 1:1 搬运。

## 交付方式
- 每个里程碑都要包含三件事：
  1. 规划更新（文档 + 验收标准）
  2. 代码实现（结构 / 样式 / 交互）
  3. 自动检查（测试 + 构建 + smoke + 像素审计）

## 里程碑

### M1 - 验收框架先行（必须先完成）
- 定义可量化验收矩阵（布局、视觉、交互、性能）。
- 提供自动化像素审计脚本：检查关键节点尺寸与排版 token（允许微小容差）。
- 产出基线截图路径规范：`docs/pixel-clone/baseline/*.png`。

### M2 - 骨架像素对齐（Layout Pass）
- 顶栏、左栏、主舞台、输入 Dock 的几何尺寸对齐。
- 空态与消息流布局对齐（assistant 文档流 + user 右侧胶囊）。
- 响应式断点行为对齐（>=1280、1024~1279、<1024）。

### M3 - 视觉像素对齐（Visual Pass）
- 颜色、边框、圆角、阴影、字体层级对齐。
- hover/focus/active/disabled/streaming 状态对齐。
- 降噪：移除影响“官网感”的多余提示和视觉装饰。

### M4 - 交互像素对齐（Interaction Pass）
- 输入区完整键鼠交互：Enter / Shift+Enter / Esc / 发送中态。
- 侧栏、命令面板、协作抽屉的开关动画与可达性。
- 聊天滚动、回到底部、新消息提示行为一致。

### M5 - 最终封板（Release Pass）
- 全量自动化通过：frontend/backend/core + smoke。
- 像素审计报告全绿。
- 基线对比通过（`ui:pixel:diff` 全绿）。
- 输出最终截图包 + 验收报告。

## 约束与原则
- KISS：仅做“对齐官网体验”必须项，不做额外功能扩张。
- YAGNI：不提前实现未验收项。
- DRY：样式 token 与布局规则集中管理，避免重复覆盖。
- SOLID：结构改动优先组件职责清晰，避免把逻辑塞进 CSS hack。
